一度公開されたアプリは、GitHubに変更をpushするだけで自動的に更新されます。
宮崎に頼まなくても、自分で改良を続けられます。

PHASE 1
アプリが公開されるまでの流れ（振り返り）
📖 参考情報
1
🗺️ ここまでにやったこと
すでに完了している内容の振り返りです。次に別のアプリを作るときの参考にしてください
1
スタッフが、富喜屋のテンプレートリポジトリ（https://github.com/Fukiya-AI/project-template）を開き、緑の「Use this template」から新しいリポジトリを作成する
Owner: Fukiya-AI ／ 公開設定は原則 Public（無料プランのPrivateアプリ枠制限のため）
2
スタッフが自分のパソコンで、そのリポジトリを git clone する（デスクトップの「AIの作業場」フォルダ内に自動でコピーされる）
3
クローンしてできたフォルダをVS Codeで開き、Claude Codeと一緒にStreamlitアプリを作成・編集する
4
スタッフが git add → git commit → git push で、作ったアプリをGitHubに送る
5
宮崎が Streamlit Community Cloud で「Create app」から対象リポジトリを選び、デプロイする
6
必要に応じて、公開範囲をメールアドレス限定に制限する
7
完成したアプリのURLを共有し、みんなが使えるようになる
✅
この方法（テンプレートから自分で作成→クローン）が推奨です。
自分でリポジトリを作成すると、作成した本人に自動的に書き込み権限（Write）が付くため、宮崎から別途権限を付与してもらう手間がありません。
また、テンプレートには.gitignoreなど秘密情報を守る設定があらかじめ入っています。
⚠️
テンプレートを使わず、いきなり自分のパソコンだけでアプリを作り始めてしまった場合は、GitHub側にリポジトリが存在しないため、後から空リポジトリの作成・権限付与など余計な手間が発生します。
次に新しいアプリを作るときは、必ず①のテンプレートから始めてください。
🔁
この「①〜⑦」は、新しいアプリを初めて公開するときだけ必要な手順です。
一度公開してしまえば、次からはPHASE 2の手順だけで改良を続けられます。
PHASE 2
完成後、自分でアプリを改良する
⏱ 毎回このくり返し
1
✏️ コードを直す
VS CodeとClaude Codeを使って、アプリの中身を修正します
streamlit run app.py
⚠️
確認を終えたら、ターミナルで Ctrl + C を押して一度止めてから次に進んでください。
2
🚀 GitHubに変更を送って公開を更新する
Claude Codeにお願いするだけでOKです
修正したものをGitHubにアップロードして、公開中のアプリに反映して
🤖
これだけでOKです。Claude Codeが git add ／ git commit ／ git push を代わりに実行してくれます。
GitHubへのpushをStreamlit Cloudが自動で検知し、1〜2分ほどで公開中のアプリに反映されます。
アプリのURLを開いて、ブラウザを再読み込み（Ctrl+Shift+R）すれば変更が確認できます。
git add .
git commit -m "直した内容を短く書く"
例）git commit -m "ラベルの文字サイズを大きくした"
git push
3
🩹 困ったときは
よくあるつまずきポイント
⚠️
「パスが存在しません」「not recognized」などのエラーが出た場合
エラーメッセージをそのままコピーして、宮崎に送ってください。一緒に解決します。
🔒
「Permission denied」「write access」など権限に関するエラーが出た場合
GitHub側の書き込み権限が外れている可能性があります。宮崎に連絡してください（本人では直せません）。
📦
新しいライブラリ（pandasなど）を使い始めた場合
requirements.txt にそのライブラリ名を追記してからpushしてください。書いていないとStreamlit Cloud側でエラーになります。

🎉 これで改良サイクルは完了です

「直す → git add / commit / push」をくり返すだけで、
アプリはどんどん育てていけます。
